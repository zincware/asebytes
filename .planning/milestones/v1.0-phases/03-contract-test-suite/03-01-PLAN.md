     - `_zarr_ragged_path(tmp_path)` -> `str(tmp_path / "test.zarr")`

Redis gets `pytest.mark.redis` (Redis is blob-level, so it goes through adapter -- include it but only with basic marks, no supports_* since ASEIO through Redis adapter may not round-trip Atoms fields perfectly; actually Redis goes BlobIO -> adapter -> ObjectIO -> ASEIO, so it should work. Give Redis all 4 capability marks + `pytest.mark.redis`). H5MD gets all 4 capability marks.
     - `OBJECTIO_BACKENDS`: same 9 backends with same marks.
     - `BLOBIO_BACKENDS`: same 9 backends with same marks. All can serve as blob backends via adapter wrapping.
   - Parametrized fixtures:

     - `objectio(tmp_path, request)` -> `ObjectIO(factory(tmp_path))`
     - `blobio(tmp_path, request)` -> `BlobIO(factory(tmp_path))`
   - `assert_atoms_equal(actual, expected, *, rtol=1e-7, atol=0)` helper function comparing: len, numbers, positions, cell, pbc, info keys (np.testing for arrays, == for scalars), arrays keys, calc results, constraints count.
   - MongoDB fixture teardown: for mongodb URI, after yield, call `remove()` on the facade's backend to clean up. Use a wrapper approach: the `aseio` fixture should handle cleanup for mongodb/redis by checking if path starts with "mongodb://" or "redis://".
