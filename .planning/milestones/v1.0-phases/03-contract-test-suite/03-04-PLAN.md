1. Update `tests/contract/conftest.py` to add read-only backend support:

   - Add a helper function `_write_ase_file(tmp_path, filename, atoms_list)` that writes a list of Atoms to a file using `ase.io.write`. This pre-populates test files for read-only backends.

   - Add factory functions for read-only backends:
     - `_traj_path(tmp_path)` -> writes s22 fixture data to `tmp_path / "test.traj"` using `ase.io.write`, returns path string
     - `_xyz_path(tmp_path)` -> writes s22 fixture data to `tmp_path / "test.xyz"`, returns path string
     - `_extxyz_path(tmp_path)` -> writes s22 fixture data to `tmp_path / "test.extxyz"`, returns path string

   IMPORTANT: The read-only factory functions need access to atom data to pre-populate files. Use a closure approach: define a `_make_readonly_factories(atoms_list)` function that returns the factory functions, or pass atom data through the fixture. The cleanest approach: the `readonly_aseio` fixture itself should accept `s22` fixture, pre-populate the file, then yield `ASEIO(path)`.

   - Add `READONLY_ASE_BACKENDS` param list:
     ```python
     READONLY_ASE_BACKENDS = [
         pytest.param(".traj", id="traj"),
         pytest.param(".xyz", id="xyz"),
         pytest.param(".extxyz", id="extxyz"),
     ]
     ```

   - Add `readonly_aseio(tmp_path, request, s22)` fixture:
     - Takes the file extension from `request.param`
     - Writes s22 atoms to `tmp_path / f"test{ext}"` using `ase.io.write(path, s22)`
     - Returns `ASEIO(str(path))` (will be readonly automatically since registry has writable_cls=None)

   - Add `hf_aseio` fixture marked with `@pytest.mark.hf`:
     - Uses a small, publicly available HuggingFace dataset that does NOT require auth
     - Use `hf://datasets/schnet/qm9` or similar small public dataset
     - Returns `ASEIO("hf://datasets/schnet/qm9")` or appropriate URI
     - If no guaranteed-public dataset exists, use a mock approach: create a fixture that builds a HuggingFaceBackend with a synthetic datasets.Dataset object directly, bypassing network. This is safer for CI.
     - PREFERRED APPROACH (no network): Create a synthetic `datasets.Dataset` from s22 fixture data, wrap it in HuggingFaceBackend manually, then wrap in ASEIO. This avoids auth walls entirely (TEST-06).

2. Create `tests/contract/test_readonly_contract.py`:

   ASE format tests (parametrized over .traj, .xyz, .extxyz):
   - `test_len(readonly_aseio)`: Assert `len(readonly_aseio) == 22` (s22 has 22 frames). Note: ASEIO on a read-only backend may need to call count_frames() or the facade may handle it. Check if `len()` works directly -- if ASEReadOnlyBackend raises TypeError for unknown length, the ASEIO facade should handle counting. Test that len returns 22.
   - `test_getitem_by_index(readonly_aseio)`: Access `readonly_aseio[0]`, verify it returns an `ase.Atoms` object with correct number of atoms.
   - `test_getitem_negative_index(readonly_aseio)`: Access `readonly_aseio[-1]`, verify it returns Atoms.
   - `test_slice(readonly_aseio)`: Access `readonly_aseio[0:5]`, verify returns list of 5 Atoms.
   - `test_iteration(readonly_aseio)`: Iterate `for atoms in readonly_aseio`, count == 22, each is Atoms.
   - `test_keys(readonly_aseio)`: Call `readonly_aseio.keys(0)`, verify returns list of strings containing at least "numbers" and "positions".
   - `test_write_rejected(readonly_aseio)`: Verify that write operations raise an appropriate error:
     - `readonly_aseio.extend([simple_atoms])` should raise (TypeError or AttributeError or similar)
     - `readonly_aseio[0] = simple_atoms` should raise
   - `test_positions_preserved(readonly_aseio, s22)`: Read back frame 0, compare positions with `s22[0].positions` using `np.allclose`. Note: .xyz format may lose precision, use `atol=1e-6`.
   - `test_numbers_preserved(readonly_aseio, s22)`: Read back frame 0, compare `get_atomic_numbers()` with `s22[0].get_atomic_numbers()`.

   HuggingFace tests (all marked `@pytest.mark.hf`):
   - `test_hf_len(hf_aseio)`: Assert len > 0.
   - `test_hf_getitem(hf_aseio)`: Access [0], verify returns Atoms.
   - `test_hf_slice(hf_aseio)`: Access [0:3], verify returns list of 3 Atoms.
   - `test_hf_iteration(hf_aseio)`: Iterate first 5 items (break after 5 to avoid loading entire dataset), verify each is Atoms.
   - `test_hf_keys(hf_aseio)`: Call keys(0), verify returns list of strings.
   - `test_hf_write_rejected(hf_aseio)`: Verify extend raises error.

   IMPORTANT: All test data must be synthetic. s22 comes from ASE collections (no auth). HuggingFace fixture must not require auth (TEST-06). If using synthetic datasets.Dataset, that satisfies TEST-06 perfectly.

   IMPORTANT: For `.xyz` format, ASE only writes/reads positions and atomic numbers (no cell, pbc, info, calc). Tests should only assert on positions and numbers for .xyz. For `.extxyz`, more fields survive. For `.traj`, full round-trip. Use the `assert_atoms_equal` helper where appropriate, but be aware of format limitations.
