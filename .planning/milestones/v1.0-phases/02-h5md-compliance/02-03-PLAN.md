  Accept: file, file_handle, file_factory, group, readonly, compression, compression_opts, chunk_frames, pbc_group=True, author_name="N/A", author_email=None.
  - Create H5MDStore(path=file, file_handle=file_handle, file_factory=file_factory, group=group, readonly=readonly, compression=compression, ...)
  - Pass store to super().__init__(store=store, group=group, readonly=readonly, ...)
  - self._pbc_group = pbc_group
  - self._author_name = author_name
  - self._author_email = author_email
  - self._h5md_initialized = False (tracks whether H5MD skeleton exists)
  - self._variable_shape: bool | None = None (auto-infer)
  - self._fixed_n_atoms: int | None = None
  - Do NOT accept variable_shape as parameter -- always auto-infer per CONTEXT.md decision

  **_init_h5md():**
  Called on first extend(). Creates the mandatory H5MD skeleton:
  - /h5md group with attrs: version=[1,1]
  - /h5md/author with attrs: name=self._author_name, email=self._author_email (if provided)
  - /h5md/creator with attrs: name="asebytes", version=get_version()
  Access the underlying h5py.File from self._store._file (H5MDStore exposes it).

  **extend() override:**
  1. If not self._h5md_initialized, call _init_h5md(), set flag
  2. Determine n_atoms per frame from the data batch
  3. Auto-infer variable_shape logic (from RESEARCH.md):
     - First batch: if all same n_atoms, set _variable_shape=False, _fixed_n_atoms=count
     - First batch: if varying, set _variable_shape=True
     - Subsequent batch: if _variable_shape=False and new batch has different counts, upgrade:
       set _variable_shape=True, call _upgrade_to_variable_shape() (resize+pad existing data)
  4. Handle connectivity separately: extract "arrays.bonds" and "arrays.bond_orders" from data,
     write them to /connectivity/{grp}/ with step/time/value structure and particles_group reference.
     Do NOT pass connectivity through regular extend path.
  5. Handle constraints: for each frame, serialize constraints to JSON via _serialize_constraints(),
     add as "info.constraints" column in the data dict before calling super().extend()
  6. Call super().extend(data) for all regular columns

  **_discover_variant() override:**
  Extend PaddedColumnarBackend._discover_variant() to also:
  - Read _h5md_initialized state (check if /h5md group exists)
  - Read _variable_shape state from stored metadata
  - Read connectivity column info

  **_serialize_constraints(atoms) / _deserialize_constraints(json_str):**
  Static/class methods using ASE's todict()/dict2constraint():
  ```python
  @staticmethod
  def _serialize_constraints(constraints):
      if not constraints: return ""
      return json.dumps([c.todict() for c in constraints])

  @staticmethod
  def _deserialize_constraints(json_str):
      if not json_str: return []
      from ase.constraints import dict2constraint
      return [dict2constraint(d) for d in json.loads(json_str)]
  ```

  **get() override:**
  Call super().get(idx) to get the base dict, then:
  - If "info.constraints" in result: deserialize JSON to constraint objects
  - If connectivity exists: read bonds/bond_orders and add to result
  - Rename "info.constraints" key appropriately

  **_upgrade_to_variable_shape():**
  When auto-infer detects variable shape after first batch was fixed:
  - Use self._resize_per_atom_axis1() (inherited from PaddedColumnarBackend) if max_atoms needs growth
  - Update _n_atoms metadata to reflect actual atom counts
  - Write _variable_shape=True to store metadata

  - All duplicated _postprocess logic

  - Manual column classification
  - Manual store management (open/close h5py files)
  - Manual metadata caching

  **Keep the helper functions at module level that are still needed:**
  - _serialize_constraints / _deserialize_constraints (move to methods)
