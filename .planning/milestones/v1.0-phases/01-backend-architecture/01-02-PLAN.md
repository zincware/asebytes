<objective>
Create PaddedColumnarBackend that stores per-atom data as padded rectangular arrays with NaN/zero-fill.

Purpose: This is the other half of the columnar backend split. While RaggedColumnarBackend (Plan 01) uses offset+flat storage, PaddedColumnarBackend pads per-atom arrays to (n_frames, max_atoms, ...) shape. This matches the storage strategy used by znh5md and is needed for H5MD compatibility in Phase 2.

Output: `_padded.py` with PaddedColumnarBackend, comprehensive tests in `test_padded_backend.py`
</objective>
