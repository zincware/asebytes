  <action>
1. **Move helpers to `columnar/_utils.py`:** Copy `concat_varying`, `get_fill_value`, `get_version`, `jsonable` from `_columnar.py` into new `columnar/_utils.py`. Keep `_columnar.py` as a re-export shim (`from .columnar._utils import *`) so any existing imports elsewhere still work.

2. **Create `columnar/_base.py` with BaseColumnarBackend:**
   Extract from `ColumnarBackend` (989-line `_backend.py`) ALL shared logic into `BaseColumnarBackend(ReadWriteBackend[str, Any])`. This includes:
   - `__init__` with store auto-detection: detect HDF5 vs Zarr from file extension (`.h5`/`.h5p` -> HDF5Store, `.zarr`/`.zarrp` -> ZarrStore). Per CONTEXT.md, drop `.hdf5` support.
   - `_discover()` base implementation (common metadata loading, column classification)
   - `_postprocess()` with base + hook pattern: common type coercion (bytes->str, JSON decode, NaN scalar->None, string arrays), calls `self._unpad_per_atom(val, col_name)` hook for per-atom values. Add optional `n_atoms` parameter that gets passed to `_unpad_per_atom`.
   - `_unpad_per_atom(val, col_name, n_atoms)` default hook (returns val as-is)
   - `_serialize_value()`, `_is_per_atom()`, `_check_index()`, `_update_attrs()`
   - `_prepare_scalar_column()`
   - `schema()`, `keys()`, `get_many()`, `get_column()`
   - `set()`, `set_column()`, `update_many()`
   - Lifecycle: `__enter__`/`__exit__`/`close()`/`__del__`/`__len__`
   - Abstract/hook methods for subclasses: `_read_per_atom_value()`, `_write_per_atom_column()`, `_discover_variant()` (called at end of `_discover`)
   - Constants: `DEFAULT_GROUP`, `_NEVER_PER_ATOM`, `_SKIP_KEYS`

3. **Create `columnar/_ragged.py` with RaggedColumnarBackend(BaseColumnarBackend):**
   - Override `get()` with ragged-specific offset+flat read logic
   - Override `extend()` with ragged-specific offset+flat write logic
   - Override `_unpad_per_atom()` to return val as-is (ragged data is already sliced by offset+length)
   - Override `_discover_variant()` for offset/length cache setup
   - Override `_read_per_atom_value()` using offset+length slice from flat arrays
   - Override `_write_per_atom_column()` using offset+flat concatenation
   - Keep ragged-specific methods: `_offsets`, `_lengths` cache management
   - Import from `._base` and `._utils`

4. **Update `_store.py`:** Change `HDF5Store.create_array` to use `maxshape=tuple(None for _ in arr.shape)` instead of `maxshape=(None,) + arr.shape[1:]`. This enables axis-1 resize needed by PaddedColumnarBackend in Plan 02. ZarrStore should similarly allow all-axis resize.

5. **Update `columnar/__init__.py`:**
   ```python
   from asebytes.columnar._base import BaseColumnarBackend
   from asebytes.columnar._ragged import RaggedColumnarBackend
   from asebytes.columnar._store import HDF5Store, ZarrStore

   # Backwards compatibility aliases
   ColumnarBackend = RaggedColumnarBackend
   ColumnarObjectBackend = RaggedColumnarBackend

   __all__ = [
       "BaseColumnarBackend", "RaggedColumnarBackend",
       "ColumnarBackend", "ColumnarObjectBackend",
       "HDF5Store", "ZarrStore",
   ]
   ```

6. **Leave `_backend.py` in place for now** -- it will be removed in Plan 03 once registry points to the new classes. This avoids breaking imports during the transition.

Important: Do NOT modify `_registry.py` in this plan -- that is Plan 03's scope. The old `_backend.py` stays importable so the registry still works during transition.
